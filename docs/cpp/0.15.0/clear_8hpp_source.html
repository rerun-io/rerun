17 /// **Archetype**: Empties all the components of an entity.
18 ///
19 /// The presence of a clear means that a latest-at query of components at a given path(s)
20 /// will not return any components that were logged at those paths before the clear.
21 /// Any logged components after the clear are unaffected by the clear.
22 ///
23 /// This implies that a range query that includes time points that are before the clear,
24 /// still returns all components at the given path(s).
25 /// Meaning that in practice clears are ineffective when making use of visible time ranges.
26 /// Scalar plots are an exception: they track clears and use them to represent holes in the
27 /// data (i.e. discontinuous lines).
